# 07. LangChain

## 1. Post-traing(Instruction-tuning, RLHF)
### 1. Pretaining vs Posting-training
**거대 언어 모델 학습 패러다임**
<br>
- `Pre-traing(사전학습)` : 방대한 텍스트 데이터를 통해 언어와 지식을 배우는 단계<br>
- `Post-train(사후학습)` : 사람이 원하는 방식으로 대화하고, 안전하고, 유용하게 만드는 단계
<br>
<br>

**Pre-traing(사전학습)**
<br>
방대한 텍스트 데이터를 이용한 Self-supervised learning을 통해 언어 패턴, 지식등을 배움<br>
학습목표 : **다음 단어 예측(Next Token Prediction)**
**Post-training(사후학습)**
<br>
유저의 의도를 파악하고 원하는 답변이 모델이 응답하도록 사후 학습 진행<br>
대표 기법 : Instruction-tuning, RLHF, DPO, RLVR
<br><br>


### 2. Instruction-tuning
사전 학습 후 거대 언어 모델은 유저의 의도와 일치X -> 파인튜닝 진행
<br>
 `파인 튜닝` : 이미 학습된 기존 모델에 특정 작업이나 모델에 맞게 **추가로 학습**
<br>
`Instruction-tuning`: 언어모델이 사람이 내린 지시문(instruction)을 따르도록 학습하는 단계<br>
- 정답 레이블이 요구되며, 다양한 태스크를 풀 수 있도록 적응하도록 중점 
- (지시문, 응답) 쌍을 모아서 언어모델을 파인튜닝
- 새로운(모델이 학습하지 않은) 태스크에서 평가를 진행

ex) Super-Natrullnstructions(데이터 셋은 1.6k+ 태스크와 3M+)
    MMLU(Masssive Multitask Language Understanging, 57개의 지식을 요구하는 태스크에서 언어모델의 성능 평가하는 벤치마크)
<br>
**->Instruction-tuning에는 Pre-training에 비해 많은 데이터 필요 X**
<br>
<br>
### 3. Reinforcemen Learning from Human Feedback(RLHF)
**Instruction-tuning한계**<br>
1. 개방형/창의적 생성과 같은 태스크 정답 존재X
2. 언어 모델링은 모든 토큰 레벨 오류를 동일 취급(오류마다 다르게 작용)
3. 사람이 만든 답변(정답 레이블) 최적 아닐 수 있음

**파이프라인 큰 그림**<br>
`입력 -> LLM -> 출력 -> 사람들 평가 -> 데이터 비교
 -> 학습 -> reward model(보상) -> 강화학습 -> LLM`
