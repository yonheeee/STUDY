# 08. Agent 모델

## 1. AI Agent
`Perception AI(지각 AI)`: 특화된 분야에서 해석 및 분석
<br>
`Generative AI(생성형 AI)`: 콘텐츠 제작(텍스트, 이미지 등)
<br>
`Agentic AI(에이전트)`: 능동적인 작업 수행("자율형 문제 해결 시스템")
<br>
`Physical AI(물리적 AI)`:AI 시스템이 물리적 실체 얻음
<br><br>
**에이전트**
: 특정 작업 수향, 결정 내리고, 합의를 도출하는 것 목표<br>
**AI에이전트**
: AI를 사용해 사용자 대신해 목표를 추구하고 작업을 수행하는 소프트웨어 시스템
<br><br>
## 2. Multi-Agent System
`대규모 언어 모델(LLM)`: 대규모 언어 모델의 최신 성과
<br>
개별 LL의 본질적인 한계<br>
- 환각 현상 : 부정확하거나 조작된 정보 생성
- 자기회귀적 성격 : 천천히 생각하거나 신중한 추론을 할 수 없음
- 스케일링 법칙 : 모델 크기 증가에 따라 성능 항상 둔화
<br><br>

집단 지성
- 지능형 에어전트들이 팀을 이뤄 지식 나누고, 문제 해결에 초점(수평적 확장)
- AI에이전트들이 서로의 장점과 시작 결합해 효울적 협업 

#### LLM기반 Multu Agent System<br>
: LLM기반 다중 에이전트 시스템 협업 메커니즘(협업 유형, 전략, 소통 등)<br>
- 문제점 <br>
집단적 추론, 의사결정 등 다중 에이전트 시스템의 발전과정에서 해결해야할 문제들이 존재<br>

구성요소<br>
- 에이전트(핵심 행위자)
- 환경(외부 세계)
- 상호작용(소통)
- 조직(계층 구조 관리 or 자발적 행동)

<br>

#### 협업 유형
- **협력**<br>
에이전트들이 개별 목표를 공동의 목표로 일치, 상호 이익이 되는 결과 달성 위해 협력<br>
장점 : 전문 분야 내 특정 세부 작업 활용 가능 <br>
단점 : 빈번한 토신과 다중 협업 채넛은 계산과 복잡성 증가, <br> 개별 에이전트 신뢰성과 성능에 크게 좌우, 한 개 이상의 에이전트 오류 시스템 전체 부정적 영행(ex.무한 루프 대화, 환각 현상 증폭)
<br>
- **경쟁**<br>
에이전트들 개별 목표 우선시, 다른 에이전트들 목표와 충돌하거나 대립할 수 있는 경쟁 요소 도입<br>
장점 : 고도의 추론 능력과 창의적 문제해결 방법을 개발<br>
에이전트 능력 시험해 시스템 적응성 강화<br>
단점 : 프롬프트를 가진 단일 에이전트에 의해 따라잡힐 수 있음
- **협력과 경쟁의 조합**<br>
  공동 목표를 위해 협력 and 다른 작업에서는 경쟁
- **전문가 홉합 모델**
  전문가 모델 최종 결과물 기여하려고 경쟁

<br>

#### 협업 전략
- **규칙 기반 프로토콜**
  - 사전에 정의된 규칙  
  에이전트 간 상호작용은 정의된 규칙 기반 통제
  - 인간의 협력 방식 모방
    규칙 기반 전략은 특정 규칙 쉽게 연결할 수 있으므로 구현과 디버깅 용이<br>
    합의 도출, 경로 찾기와 같이 절차 명확, 변동성 적은 작업 효율적
- **역할 기반 프로토콜**
  - 사전에 정의된 역할
    세분화된 목표를 기반 역할 분담
  - 주로 도메인 지식 기반
    각 에이전트 특정 택임 할당, 역할 준수를 통해 일치성 강화<br>
    개별 모듈 재활용 가능성

<br>

#### 커뮤니케이션 구조
- **중앙집중형 구조**
  - 참여자- 서비스 제공자 구조
  - 중앙 에이전트
    LLM-블렌더는 한 라운드에서 서로 다른 LLM 호출하고, 두 개씩 비교 순위 책정 후, 최상위 응답들을 결합
- **탈중앙화 구조**
  - 에이전트에 대한 통제 및 의사결정 분산<br>
    (높은 확장성 제공, 에이전트 자율적 작동하고 시스템 변화에 적음)

<br>
